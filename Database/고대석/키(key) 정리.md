# 키(key) 정리

## **Key의 종류**

- **슈퍼 키(Super Key)**
    - **유일성**을 만족하는 키. 예를 들면, {학번 + 이름}, {주민등록번호 + 학번}
    - **어떤 속성끼리 묶던 중복값이 안나오고 서로 구별**
- **복합 키(Composite Key)**
    - **2개 이상**의 속성(attribute)를 사용한 키.
- **후보 키(Candidate key)**
    - **유일성**과 **최소성**을 만족하는 키. 기본키가 될 수 있는 후보이기 때문에 후보키라고 불린다. 예를 들면, 주민등록번호, 학번 등
    - 슈퍼키들 중에서 속성을 최소한의 갯수로 구분할 수 있어야 후보키가 될 수 있다.
- **기본 키(Primary key)**
    - **후보키들 중에서 하나를 선택한 키로 최소성과 유일성을 만족**하는 속성이다.
    - 테이블에서 기본키는 **오직 1개**만 지정할 수 있다.
    - 기본키는 **NULL 값을 절대 가질수 없고, 중복된 값을 가질 수 없다**.
- **대체 키(Surrogate key)**
    - 후보키가 두개 이상일 경우 그 중에서 어느 하나를 **기본키로 지정하고 남은 후보키**
    들을 대체키라 한다.
- **외래 키(Foreign Key)**
    - 어떤 테이블(Relation) 간의 기본 키(Primary key)를 참조하는 속성이다. 테이블(Relation)들 간의 관계를 나타내기 위해서 사용된다.

### 문제

1. 최소성과 유일성을 만족하고 기본 키가 될 수 있는 키 ⇒ 후보 키(Candidate ****key)
2. 외래키로 참조될 열의 값은 참조될 테이블에서 무슨 키로 설정되어 있어야 하나? ⇒ 기본 키
3. 유일성을 만족하는 키 ⇒ 슈퍼 키
